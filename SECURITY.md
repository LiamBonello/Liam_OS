# Security Policy

Liam_OS is experimental and not ready for sensitive, production, or commercial use.

## Supported Versions

Only the current development tree is reviewed. Older source archives and test images are not supported.

## Reporting Issues

Do not publish exploit details for unresolved issues in public release notes. Record the issue, affected component, expected impact, reproduction steps, and proposed fix in the private project tracker until a fix is available.

## Current Security Limits

- User/kernel isolation is still incomplete.
- Persistent storage, installer, update, and signing flows are not release-grade yet.
- Driver isolation and device permission models are not complete.
- There is no stable security update channel yet.

Before any public or paid release, create a public contact address for vulnerability reports and define supported release branches.
