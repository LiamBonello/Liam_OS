# Commercial and Licensing Notes

This file is not legal advice. It identifies the issues that must be resolved before selling Liam_OS.

## Accurate positioning

If Liam_OS Desktop is built on Linux, use accurate language such as:

> Liam_OS Desktop is a custom Linux-based operating system for gamers, developers, and creators.

Avoid claims such as:

> Built completely from scratch.

unless every relevant layer is actually built from scratch.

## Open-source obligations

Before commercial distribution:

- Track every upstream component and license.
- Preserve required notices.
- Provide source code where licenses require it.
- Keep modified source availability obligations clear.
- Do not remove attribution from upstream projects.

## Proprietary software

Do not bundle proprietary software unless redistribution rights are confirmed. This includes, but is not limited to:

- Steam
- Discord
- Nvidia drivers
- Microsoft fonts/tools
- Commercial design applications

Safer approach: provide installer links or post-install setup flows where licensing permits.

## Trademarks

Avoid implying endorsement by upstream projects or companies. Use names such as Steam, Nvidia, Ubuntu, KDE, Microsoft, and Figma only in a descriptive and legally safe way.

## Required commercial documents

- End-user license / terms.
- Privacy policy.
- Update support policy.
- Refund/support policy.
- Open-source notices.
- Security vulnerability contact process.
